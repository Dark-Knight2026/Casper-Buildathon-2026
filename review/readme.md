## Storage Policy

This directory contains **permanent historical records** of code reviews. Reviews are point-in-time artifacts that document the state of the codebase at the time of review.

**Naming convention:** `YYYY_MM/backend_code_review_YYYY_MM.md`

**Update policy:** Reviews are append-only. New reviews are added as new files; existing reviews are not modified except for corrections of factual errors.
